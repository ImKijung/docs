## 설치 안내

### AWS Log 설치

1. 리전을 선택하세요. 선택된 리전이 있으면 ***설치*** 버튼이 활성화됩니다.
1. ***설치*** 버튼을 클릭하면 CloudFormation 페이지로 이동합니다. 기본 파라미터는 입력되어 있으므로 페이지 하단으로 내려가 ***승인*** 체크 및 ***스택 생성*** 버튼을 클릭하세요.
1. 파라미터와 설치 IAM 권한에 대해 자세히 알고 싶다면 설치 버튼 아래의 각 항목을 펼쳐 내용을 확인할 수 있습니다.

### IAM Role 생성

:::note
사용자가 직접 AWS IAM 서비스로 들어가 Role을 생성해야 합니다.
:::

### IAM Role ARN 및 리전 입력

리소스 조회를 위해 IAM Role과 Log Forwarder가 설치된 리전을 입력받습니다. 중간에 페이지를 이탈하지 않은 경우 AWS Log Forwarder 설치 단계에서 입력한 리전이 자동으로 선택되어 있습니다.

1. Role ARN을 입력하면 ***등록*** 버튼이 활성화됩니다. 버튼을 클릭하면 구독할 수 있는 리소스 목록을 조회합니다.
1. 조회 실패 시 다음과 같은 메세지가 나타납니다. ***등록*** 버튼을 다시 누르거나, 이후 리소스 구독 단계에서 조회 목록 ***새로고침***을 통해 재조회가 가능합니다.

### Log Forwarder가 지원하는 AWS Resource Log 확인

:::note
현재 지원 중인 리소스 로그 확인을 위한 참고 단계로 설치 과정에 영향을 주지 않습니다.
:::

### AWS Resource Log 선택 및 Subscribing

**IAM ROLE ARN 및 리전 입력** 단계에서 조회한 리소스 목록 중 구독할 리소스를 선택 및 등록합니다.

1. 구독할 CloudWarch Log와 S3 bucket을 각각 선택합니다. 
:::note
복수 선택이 가능합니다.
:::
1. 리소스를 선택한 후 순서대로 AWS Role ARN과 Log Forwarder ARN을 입력합니다. 페이지를 이탈하지 않은 경우 이전 단계에서 입력한 AWS Role ARN이 자동으로 입력되어 있습니다.
1. 두 ARN을 모두 입력하면 ***등록*** 버튼이 활성화 됩니다. 버튼을 누르면 선택한 리소스를 구독 목록에 등록합니다. 등록 성공 여부는 다음 이미지와 같이 메세지로 표시됩니다.

## 모니터링 활성화

설치를 완료한 뒤 설치 페이지 하단에 ***모니터링 서비스 활성화*** 버튼을 클릭해 Log Forwarder 메뉴를 활성화합니다.

***모니터링 서비스 활성화*** 버튼을 클릭하면 다음과 같이 왼쪽 메뉴에 로그 메뉴가 활성화됩니다.
