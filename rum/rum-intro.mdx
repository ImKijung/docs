## Real User Monitoring(RUM)이란?


실 사용자 모니터링(Real User Monitroing, 이하 **RUM**)이란 웹 서비스의 End-to-End 서비스 응답시간 측정을 통해 최종 사용자 측면의 서비스 체감 상황을 모니터링합니다. 또한 웹 서비스의 실시간 성능 분석 및 가용성 관리를 통해 장애 감지, 예방 체계를 지원합니다.

> "49%의 고객이 **2초 이하의 로딩 속도를 기대**하고, **18%의 고객은 1초도 지체 없는 로딩 속도를 기대**한다는 2년 전 자료가 있다.", "**로딩 속도 1초**가 빨라지면 아마존 판매량이 **1% 증가**하고 구글 검색량 **0.2% 증가**, 월마트의 전환율이 **2% 증가**한다"
>
> \- <측정 가능해야 관리할 수 있다(What gets measured gets managed.)> 측정의 중요성을 역설한 피터 드러커

> 아마존의 경우 로딩 시간이 0.1초 증가할 때마다 매출이 1%씩 하락했고, 구글은 페이지 로딩 시간 0.5초 증가로 인해 매출 및 트래픽이 20% 감소했다. 로딩 시간이 매출과 트래픽으로 직결됨을 증명한 것이다.
>

> 소비자는 그 어느 때보다 요구 사항이 많으며, 여러분의 사이트 경험을 평가할 때 경쟁업체와 비교할 뿐만 아니라 매일 사용하는 동급 최고의 서비스와 비교하며 여러분을 평가합니다.

> \- [https://web.dev/why-speed-matters/](https://web.dev/why-speed-matters/)

## RUM, 왜 필요한가?

와탭의 RUM 서비스를 통해서 웹 애플리케이션 운영자는 다음과 같은 기대효과를 가질 수 있습니다.

* 신규 웹 서비스를 오픈하기 전, 웹 서비스의 사용자 측면에서 이상 유무에 대한 테스트 도구로 활용할 수 있습니다.
* 사용자가 체감하는 웹 서비스의 성능 관리를 통해 고객 불만을 보다 빠르게 인지, 서비스 개선이 필요한 서비스를 도출하는 데에 활용할 수 있습니다.
* 응답시간을 줄이기 위한 최적화 정보를 제공합니다.
* 최종 사용자의 애플리케이션 경험에 영향을 미치는 요소들을 파악합니다.
* 웹 애플리케이션에 접속한 최종 사용자의 지역 및 환경 별 성능을 분석합니다.
* 브라우저 영역에서의 성능 이상을 탐지합니다.

## 주요 기능

브라우저 **RUM**(Real User Monitoring)은 최종 사용자 입장에서 웹사이트 이용의 문제 여부, 최종 사용자의 사용성, 최종 사용자의 환경을 모니터링합니다. 다음의 주요 기능을 제공합니다.

* 다음의 주요 지표를 측정할 수 있습니다.
  * Page load 응답시간
  * Route change 응답시간
  * AJAX 응답시간
  * Resource 응답시간
  * Web vitals 수치
  * Java script error

* 사용자 접속 환경을 분석하여 사용자 브라우저, 운영체제, 디바이스 정보를 제공합니다.

* 사용자 IP를 분석하여 국가, 도시, 지역 정보를 제공합니다.

* 브라우저 에이전트가 발급한 SessionID를 분석하여 현재 웹 사이트를 사용 중인 동시 사용자 수를 제공합니다.

* 서버, 애플리케이션, 데이터베이스 등 와탭의 다른 서비스와 함께 대시보드를 구성하여 실시간으로 현황을 파악할 수 있습니다.

## 브라우저 모니터링 시작하기

다음의 단계를 따라 브라우저 모니터링을 시작할 수 있습니다.

1. 와탭 계정이 없다면 [회원 가입](https://www.whatap.io/ko/signup/)을 진행하세요.

1. 브라우저 호환성 및 사전 설정을 살펴보세요.

1. [와탭 모니터링 서비스](https://service.whatap.io)로 이동해 브라우저 프로젝트를 생성하세요.

1. 브라우저 에이전트를 적용하세요.
