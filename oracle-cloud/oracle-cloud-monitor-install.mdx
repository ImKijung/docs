와탭 Oracle Cloud Monitor 모니터링은 API 연동을 통해 metric을 주기적으로 수집합니다. 수집을 원하는 서비스에 해당하는 Oracle Cloud Monitor 네임스페이스를 5분 기간으로 원하는 stat을 취사선택해 수집합니다.  모니터링 시작 시 모든 리전의 사용자 자원을 검색하여 원클릭으로 모니터링을 시작할 수 있습니다. Oracle Cloud API 등록 시 필요한 ssh 공개 키를 자동 생성하여 안전하게 편리하고 사용하실 수 있습니다. 
 
{@include: ../getting-started/_create-project-accesskey-item.mdx} 

## 모니터링 절차

1. 사용자 와탭 콘솔에서 인티그레이션 프로젝트를 Oracle Cloud Monitor 사용 상태로 생성하세요.
1. Oracle Cloud 인증 정보를 입력하세요.
1. Oracle Cloud Monitoring Namespace 및 수집 주기를 선택하세요.
1. Region, Metric, Stats 등을 선택하세요.

:::note
설치 후 사용량에 따라 와탭 요금과는 별개로 클라우드에서 추가 비용이 발생할 수 있습니다. 설치 전 사용 중인 클라우드 서비스의 가격 정책을 [다음 문서](https://www.oracle.com/kr/cloud/price-list/)에서 확인하세요.
:::

## 인증 정보 입력

![Oracle Cloud Monitor](https://img.whatap.io/media/images/whatap_oci_auth_dialog.png)

1. RSA Public Key를 생성 후 메모장에 생성한 RSA Publkic Key를 복사합니다. 이후 ***RSA Public Key생성***을 클릭하면 생성된 KEY를 언제든지 확인 할 수 있습니다.
1. Oracle Cloud 콘솔 ***Identity & Security*** 메뉴 하단 ***Groups***를 선택하여 모니터링에 사용할 Group을 신규 생성하거나 선택합니다.
1. Oracle Cloud 콘솔 ***Identity & Security*** 메뉴 하단 ***Policies***를 선택하여 모니터링용 Policy를 생성합니다.
    * ***Create Policy***를 클릭하여 이름, Descrition 및 Compartment를 입력합니다.
    * ***Policy Builder***에서 ***Show Manual Editor***를 체크하여 아래 권한을 입력 후 ***Create***를 클릭 합니다.
        * Allow group {그룹} to read metrics in compartment {컴파트먼트}
        * Allow group {그룹} to inspect metrics in compartment {컴파트먼트}
1. Oracle Cloud 콘솔 ***Identity & Security*** 메뉴 하단 ***Users***를 선택하여 User를 생성하거나 선택 합니다.
    * ***Create User***를 클릭하여 신규 User 를 생성하거나 ***User***를 선택하여 클릭합니다.
    * User Details 화면에서 해당 유저를 2에서 선택한 Group에 추가합니다.
    * 왼쪽 하단의 ***API Keys***를 클릭합니다.
    * ***Add API Key*** 버튼을 클릭합니다.
    * 라디오 버튼의 ***Paste Public Key***를 선택하여 2단계에서 받은 RSA Public Key를 입력합니다.
    * 추가 완료 시 user, tenancy 정보를 화면에서 확인할 수 있습니다. 이 정보를 메모장에 복사합니다.
1. Oracle Cloud 콘솔 ***Identity & Security*** 메뉴 하단 ***Compartments***를 클릭하여 모니터링할 Compartment의 OCID를 확인 및 메모장에 복사 합니다.
1. 4에서 확인한 Tenancy와 User 및 5에서 확인한 CompartmentID를 와탭에 입력합니다.
