이 문서에서는 와탭 모니터링 서비스 중 하나인 서버 모니터링의 ***리소스 보드*** 메뉴에서 차트형 위젯에 대한 분석 방법을 소개합니다. 서버(Server)의 문제를 ***리소스 보드***를 통해 어떻게 파악하고 분석하는지 살펴보겠습니다. 리소스 보드의 기능에 대한 개괄 소개는 [다음 문서](../server/resourceboard)를 참조하세요.

서버 모니터링의 핵심은 프로세스입니다. 프로세스가 정상 동작 범위 내에 중단없이 운영될 수 있도록 모니터링을 통해 서버 현황을 빠르게 파악하고 각종 장애 상황에 대응할 수 있어야 합니다. 리소스 사용량을 조회해 추론하는 것이 그 시작입니다. 와탭의 ***리소스 보드***는 리소스 사용량을 데이터 집약적인 화면에서 한눈에 확인할 수 있도록 CPU, Memory, Disk, Network 관련 연관 지표들로 구성된 시계열 차트 위젯을 제공합니다.

* 서버 요약 지표: ***Server***, ***OS***, ***Total Cores***, ***Avg CPU***, ***Avg Memory***, ***Avg Disk***
* CPU 사용량 분포도: ***CPU Resource Map***
* 리소스 사용량 Top 5: ***CPU TOP5***, ***Memory TOP5***, ***Disk I/O TOP5***, ***Disk IOPS TOP5***, ***프로세스 CPU TOP5***, ***프로세스 TOP5***

## 서버 요약 지표

![인포 패널](/img/resourceboard-info.png)

***리소스 보드***의 상단 인포 패널을 통해 프로젝트에 등록된 전체 서버들에 대한 요약 지표들을 쉽게 확인할 수 있습니다. 

***Server*** 위젯에서 전체 서버의 수와 장애가 있는 서버의 수를 조회할 수 있습니다. ![right-arrow](/img/right-arrow.svg) 버튼을 선택하면 ***서버 목록*** 메뉴로 이동합니다. ***상태*** 컬럼에서 ![위험 아이콘](/img/ic-warning-critical.svg)<span class='vslow'>위험</span>(<span class='vslow'>빨간색</span>), ![경고 아이콘](/img/ic-warning-org.svg)<span class='slow'>경고</span>(<span class='slow'>주황색</span>) 단계의 예외 상태와 ![정상 아이콘](/img/ic-check-circle-filled.svg)<span class='snormal'>정상</span>(<span class='snormal'>녹색</span>), ![비활성 아이콘](/img/ico-disable-agent.svg)<span class='k8s-normal'>비활성</span>(<span class='k8s-normal'>회색</span>) 상태를 아이콘과 색으로 쉽게 구분하여 확인할 수 있습니다.

***OS*** 위젯은 프로젝트 내 운영체제의 수, ***Total Cores***는 전체 서버들 코어의 합, ***Avg CPU***는 전체 서버들의 CPU 평균 사용량, ***Avg Memory***는 전체 서버들의 메모리 평균 사용량, ***Avg Disk***는 전체 서버 디스크 디바이스들의 평균 사용량을 나타냅니다. 

인포 패널의 위젯을 확인하여 위와 같이 전체 리소스 사용량의 윤곽을 빠르게 살필 수 있습니다. 한 예로 위의 예시 화면 속 평균 CPU 사용량은 4%로 리소스 과다 투입 상태라는 것을 알 수 있습니다.

:::note
* ***서버 목록***에 대한 자세한 설명은 [다음 문서](../server/server-list)를 참조하세요.
* ***리소스 보드*** 중앙 차트의 ![서버상태맵 아이콘](/img/dashboard-honeycomb.svg) 아이콘을 선택해 ***Server Status Map***으로 이동하면 벌집 모양의 맵을 통해 프로젝트 내 서버 상태를 시각적으로 파악할 수 있습니다.
:::

## CPU 사용량 분포도

프로세스 정상 동작을 확인하기에 가장 중요한 요소가 CPU 사용량입니다. ***리소스 보드*** 중앙 차트의 ![리소스맵 아이콘](/img/dashboard-hitmap.svg) 아이콘을 선택해 ***CPU Resource Map*** 위젯으로 이동하면 프로젝트 내 전체 서버의 CPU 사용량 분포도를 확인할 수 있습니다. 셀 영역을 드래그하면 해당 구간의 상세 정보를 조회할 수 있습니다.

![리소스맵 리소스보드](/img/resourceboard-resourcemap.png)

리소스 사용량은 과다와 과소 상태를 모두 피하고 50% 전후를 유지하는 것이 바람직합니다. 이는 효율성을 유지하기 위한 것입니다. 50% 전후는 사용량 급증 시 과부하로 인한 지연이 발생하더도 약 1분 내외로 쾌적한 상태로 회복할 수 있습니다.

* 과다 투입: 과다 투입의 경우 인프라 비용이 필요 이상으로 지출되고 있는 상태입니다. CPU 리소스맵
* 과소 투입: 과소 투입의 경우 서비스의 품질 악화 및 사용 편의성이 저하된 상태입니다.
