Java 에이전트 설치

와탭 모니터링 서비스를 사용하기 위해서는 회원 가입 후 프로젝트를 생성하고 대상 서버에 에이전트를 설치해야 합니다. 회원 가입에 관한 자세한 내용은 다음 설명서를 참조하세요.

다음 동영상 가이드를 참조하세요.

프로젝트 생성​

에이전트를 설치하려면 먼저 프로젝트를 생성해야 합니다.

  1. 모니터링 프로젝트를 생성하려면 왼쪽 메뉴에서 + Project 버튼을 선택하세요.
  2. 프로젝트 이름을 입력한 다음 이용할 플랫폼을 선택하세요.
  3. 가까운 데이터 서버 지역(AWS 또는 Azure)을 선택하세요.
  4. 타임 존을 선택하세요.
    기본 값은 (GMT +9:00) Seoul, Tokyo, Yakutsk입니다.
  5. 저장 버튼을 선택하세요.
노트
  • 와탭의 모니터링 프로젝트는 모니터링 단위를 의미합니다. 프로젝트에 관한 자세한 내용은 다음 설명서에서 확인하세요.
  • 데이터 서버 지역은 리전을 의미합니다.
  • 클라우드에서 리전이란 클라우드 서비스를 제공하기 위해 설치하는 데이터 센터의 묶음입니다.
  • 특정 리전을 선택하면 해당 리전에 속한 데이터 센터에 사용자 데이터를 저장합니다.

프로젝트 액세스 키 확인​

프로젝트 액세스 키는 와탭 서비스 활성화를 위한 고유 ID입니다.

설치 안내 섹션에서 프로젝트 액세스 키 발급받기 버튼을 선택하세요. 프로젝트 액세스 키를 자동으로 발급 받은 후 다음 단계를 진행합니다.

팁

프로젝트를 생성한 다음에는 자동으로 에이전트 설치 페이지로 이동합니다. 에이전트 설치 페이지로 이동하지 않는다면 왼쪽 메뉴에서 전체 프로젝트를 선택한 다음 새로 생성한 프로젝트를 선택하세요.

에이전트 다운로드​

프로젝트 액세스 키를 발급 받으면 와탭 에이전트 파일 다운로드 섹션에서 Download 버튼이 활성화됩니다.

  1. Download 버튼을 선택해 와탭 에이전트 파일을 다운로드하세요.

    팁

    다음 명령어를 이용해 서버에서 직접 다운로드할 수도 있습니다.

    wget https://service.whatap.io/agent/whatap.agent.java.tar.gz

  2. 모니터링 대상 서버에 접속하세요.

  3. 원하는 위치에 에이전트 파일의 압축을 해제하세요. 이 위치가 $WHATAP_HOME이 됩니다.

    tar -zxvf whatap.agent.java.tar.gz
JAVA 에이전트 설치 파일 구성

Java 에이전트 파일은 애플리케이션 모니터링에 필요한 정보를 추출해 와탭 수집 서버로 전달하는 트레이서와 트레이서가 데이터를 보낼 수 있도록 돕는 요소로 구성되어 있습니다.

  • whatap.conf
    모니터링 에이전트의 설정 정보, 프로젝트 액세스 키(또는 라이선스 키), 와탭 데이터 수집 서버의 IP 정보 등을 저장하는 설정 파일입니다. 자세한 내용은 다음 설명서를 참조하세요.

  • paramkey.txt
    화면 상에서 각종 변수를 확인할 때 필요한 파일입니다.

  • ping.sh(bat)
    현재 설정한 프로젝트 액세스 키, 와탭 데이터 수집 서버로 통신을 시도해 설정이나 네트워크 이슈가 있는지 확인하기 위한 스크립트(배치) 파일입니다.

  • proxy.sh(bat)
    whatap.agent.proxy-X.Y.Z.jar 파일을 시작하기 위한 스크립트(배치) 파일입니다.

  • resmon.sh(bat)
    CPU, Memory, Disk 정보를 정상 수집하는지 확인하기 위한 스크립트(배치) 파일입니다.

  • whatap-logsink-lz4-X.Y.Z.jar
    애플리케이션 서버의 log 정보를 수집하기 위한 plugin 형식의 파일입니다.

  • whatap.agent-X.Y.Z.jar 애플리케이션 서버의 각종 모니터링 정보를 수집 및 수집 서버에 전송하는 모니터링 에이전트 파일입니다.

  • whatap.agent.proxy-X.Y.Z.jar
    애플리케이션 서버가 동작하는 서버에서 WhaTap 측 수집 서버에 직접 연결이 불가한 경우, 그 사이의 Proxy 서버(경유 서버)에서 경유 역할을 하는 애플리케이션입니다.

  • logs/whatap-yyyymmdd.log
    모니터링 에이전트에서 출력하는 로그 파일입니다.

whatap.conf 파일 설정​

압축을 푼 $WHATAP_HOME 경로로 이동하세요. $WHATAP_HOME/whatap.conf 파일 내용에 라이선스 키와 데이터 수집 서버 주소가 정상적으로 들어가 있는지 확인하세요. 값이 없다면 와탭 모니터링 서비스 화면으로 이동해 설치 안내 섹션에서 프로젝트 액세스 키 값을 복사한 다음 붙여넣기하세요.

$WHATAP_HOME/whatap.conf
license={프로젝트 액세스 키}
whatap.server.host={수집 서버 IP}
노트
  • Download 버튼을 선택해 에이전트를 다운로드했다면 whatap.conf에 설정 값이 들어가 있습니다.
  • 서버에서 직접 다운로드했다면 프로젝트 액세스 키와 수집 서버 IP를 직접 입력하세요.
  • whantap.conf 파일에 대한 자세한 내용은 다음 설명서를 참조하세요.

JVM 옵션 추가​

  1. 적용 대상(WAS 및 Batch, Daemon과 같은 Java 애플리케이션)에 와탭 에이전트 경로를 javaagent 옵션을 추가하세요.

    JVM_OPTS
    -javaagent:/whatap/whatap.agent-X.Y.Z.jar

  2. 서버를 다시 시작하세요.

  3. 프로세스 정보에서 설정한 환경 변수를 확인하세요.

다음 이미지와 같다면 설정 적용을 완료한 것입니다.
