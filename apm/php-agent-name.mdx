와탭은 애플리케이션 서버 식별을 위해 애플리케이션 서버 종류, 서버 IP, 서비스 포트를 조합하여 고유 식별자로 사용합니다. 필요에 따라 사용자가 지정한 명칭이나 변경한 패턴을 사용하는 것도 가능합니다.

애플리케이션 서버에서 추출한 정보를 활용하는 이유는 애플리케이션 서버 정지, 네트워크 단절 또는 에이전트 문제 등으로 수집 서버와 에이전트의 통신 단절 상태가 복구되었을 때, 재접속한 에이전트에서 송신되는 정보와 기존 에이전트에서 송신된 정보의 연속성을 유지하기 위해서입니다. 

와탭이 애플리케이션 서버를 식별하기 위해 사용하는 기본 패턴은 다음과 같습니다.

* default: {type}-{ip2}-{ip3}-{process}-{dockerid}-{ips}

기본 패턴 변경은 *whatap.ini*에서 설정 가능합니다.

* object_name default: {type}-{ip2}-{ip3}-{process}-{dockerid}-{ips}

|설정|설명|
|----|----|
|type|whatap.app_name에 설정된 값을 사용합니다.|
|Ip#|Ip를 .으로 나누었을 때 #번째 자리(0부터)를 사용합니다.|
|process|whatap.app_name에 설정된 값을 사용합니다.|
|hostname|서버 호스트 명을 사용합니다.|
|dockerid|도커 컨테이너 아이디|
|ips|전체 아이피 정보 (ip4, ip6)|

* **whatap.object_name**<span class='type'>String</span><br/>
    기본값 `{type}-{ip2}-{ip3}-{process}`  
    애플리케이션을 식별하는 에이전트 이름(ONAME) 체계입니다. ONAME을 토대로 OID가 생성됩니다. 

{@include: _php-restart-whatap-service.mdx}

* **wwhatap.app_name**<span class='type'>String</span><br/>
    애플리케이션을 식별하는 에이전트 이름(ONAME) 체계에 사용되는 애플리케이션 명으로 object_name의 {type}에 해당하는 값입니다.

{@include: _php-restart-whatap-service.mdx}

* **whatap.app_process_name**<span class='type'>String</span><br/>
    애플리케이션을 식별하는 에이전트 이름(ONAME) 체계에 사용되는 애플리케이션 프로세스 명으로 애플리케이션 서버의 CPU, Heap Memory 등을 수집할 대상 프로세스를 설정합니다. object_name의 {process}에 해당하는 값입니다.

{@include: _php-restart-whatap-service.mdx}
