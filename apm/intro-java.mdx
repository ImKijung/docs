## 모든 JAVA 모니터링

와탭은 HTTP 프로토콜을 서비스하는 대부분의 서비스 애플리케이션을 모니터링할 수 있으며 내부의 장애를 찾아낼 수 있습니다. **Tomcat**, **WebLogic**, **JEUS** 등 Web Application Server(이하 WAS), J2EE 기반이 아닌 서버에 대해서도 모니터링이 가능합니다. 뿐만 아니라 데몬 형태가 아닌 Batch 형태의 자바 애플리케이션에 대해서도 성능 모니터링이 가능합니다.

## 트랜잭션 추적

WAS는 브라우저의 요청을 받아 HTML/JSON 형태의 응답을 리턴합니다. 이런 서비스형 애플레이션에서 요청과 응답을 우리는 트랜잭션이라 부르고 있습니다.

자바 환경의 애플리케이션 서버에서 와탭 에이전트를 설치하면 먼저 클라이언트(브라우저)의 요청이 추적되는지 확인할 수 있습니다.

:::note
일반적인 WAS는 특별한 설정없이 트랜잭션 추적이 가능하도록 되어있습니다. 하지만 새로운 기술을 적용하는 애플리케이션에서는 추적이 안되는 경우가 있는데 이 경우는 추가 설정이 필요할 수 있습니다.
:::

## 트랜잭션 프로파일링

트랜잭션(요청과 응답)을 수행하는 동안 애플리케이션은 데이터베이스를 액세스하거나 외부 시스템의 정보를 추가로 요청할 수 있습니다. 이때 와탭 에이전트는 트랜잭션 수행 중 사용하는 SQL 문의 성능 혹은 외부 HttpCall의 성능을 추적합니다.

이것을 우리는 트랜잭션 프로파일링이라고 부릅니다.

## 성능 매트릭스 확인

웹 애플리케이션의 트랜잭션 추적과 트랜잭션 처리에 대한 처리량, 응답성능, 자바 프로세스의 주요 자원(Heap, Gc 등) 사용량 그리고 OS 환경의 시스템 CPU, 메모리 등의 주요 메트릭스를 모니터링할 수 있습니다.
