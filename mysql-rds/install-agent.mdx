MySQL RDS는 DB 접속만 할 수 있다면 별도의 설치 과정이 필요하지 않습니다. [회원 가입](https://www.whatap.io/ko/signup/) 후 프로젝트를 생성한 다음 화면의 안내에 따라 진행하세요. 회원 가입에 관한 자세한 내용은 [다음 문서](account/account-manage)를 참조하세요.

```bash showLineNumbers
create user whatap identified by 'DB_PASSWORD'; ## 원하는 password를 입력하세요.
grant process on *.* to whatap;
grant select on performance_schema.* to whatap;
grant show databases on *.* to whatap;
 
## 테이블 정보를 수집할 경우
## table list를 가져올 db명.* 또는 *.* 지정해야 table 정보를 수집합니다.
grant select on DB명.* 또는 *.* to whatap;
 
## replication 사용시
grant replication client on *.* to whatap;
grant replication slave on *.* to whatap;
 
## kill session 사용시
grant super on *.* to whatap;
```

## DB 정보 입력

모니터링할 대상 DB와 연결하기 위한 필수 정보를 차례로 입력하세요. <span class='vslow'>*</span> 표시는 필수 입력 항목입니다.

![DB 정보 입력](/img/mysql-rds-db-info.png)

* ***인스턴스 명***: DB를 식별하기 위한 명칭을 입력하세요. 프로젝트 내에서 고유한 명칭이어야 하며, 영어 소문자, 숫자, 붙임표(-)만 입력할 수 있습니다.

* ***DB IP***: DB 서버의 IP 주소를 입력하세요.

* ***DB Port***: 와탭 수집 서버와 연결할 포트 번호를 입력하세요.

* ***DB User ID***: 앞서 생성한 모니터링용 계정을 입력하세요.

* ***DB Password***: 앞서 생성한 모니터링용 계정의 비밀번호를 입력하세요.

모든 항목을 입력했으면 ***완료*** 버튼을 선택하세요. 와탭 수집 서버(DBCTL)에서 접속을 시도합니다. 이 과정에서 내부적으로 에이전트를 생성한 후 입력한 주소와 계정 정보로 접속을 시도하기 때문에 다소 시간이 걸릴 수 있습니다.
