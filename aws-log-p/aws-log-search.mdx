홈 화면 > 프로젝트 선택 > ***대시보드*** > ***로그 검색***

와탭 모니터링 서비스 초기 화면에서 프로젝트를 선택한 다음 프로젝트 메뉴 하위에 ***대시보드*** > ***로그 검색*** 메뉴를 선택하세요. 

***로그 검색*** 메뉴에서 통합 수집된 대량의 로그를 다양한 조건으로 검색하고 사용자가 원하는 로그를 특정할 수 있습니다. 복수의 검색 조건을 파싱된 키와 밸류로 지정할 수 있어 원하는 조건에 일치하는 로그 데이터만 추출합니다. 

동적 페이지로 검색된 로그 데이터를 정해진 라인 단위로 가져오며, 스크롤 등에 의해 하단에 닿으면 자동으로 다음 데이터를 가져와 표시합니다. {@include: ../common-items/_log-ui-main-term.mdx}

![로그 검색 sc](/img/aws-log-p/log-search.png)

## 데이터 조회하기 

* 스크롤이 바닥에 닿으면 다음 데이터를 조회합니다. 한 번에 30,000개의 로그를 조회합니다.
* ![number-s](/img/number-03.png) 로그 테이블에서 로그 데이터를 시간 순과 역순으로 조회할 수 있습니다. ***Timestamp*** 컬럼 기준 ***최근 순***과 ***과거 순*** 중 원하는 조회 방식을 선택하세요.

## 로그 Content 확인하기

:::note 

**Content란?**

Content는 로그 메시지를 의미합니다.

:::

* 로그 컬럼의 첫 번째 줄은 로그의 파싱(parsing)된 키와 값이고 두 번째 줄은 로그의 content입니다.

* ![number-s](/img/number-03.png) ***테이블***의 행(로그)마다 ![아이콘](/img/ic-caret-right.svg)***더보기*** 버튼이 있습니다. ![아이콘](/img/ic-caret-right.svg)***더보기*** 버튼을 선택하면 ![number-s](/img/number-04.png)처럼 해당 로그의 전체 content를 확인할 수 있습니다.
* 로그의 태그를 선택하면 ***복사***, ***검색***, ***제외 검색***을 할 수 있는 드롭다운 메뉴가 나타납니다.

## 필터

#### 필터 적용

![number-s](/img/number-01.png) 왼쪽 ***시간 선택창***에서 시간 범위를 지정할 수 있습니다. 오른쪽에서 필터를 적용하면 입력한 조건에 맞는 로그를 필터링합니다. 복수의 필터를 입력할 수 있습니다. 필터의 태그가 같은 경우 OR(`||`)로, 그렇지 않은 경우는 AND(`&&`)로 적용됩니다. 

입력창에 값을 직접 입력하거나 ***필터*** 입력창을 클릭해 필터를 지정할 수 있습니다. 입력창에 값을 입력할 때 띄어쓰기로 키워드를 구분합니다. 키워드 내 띄어쓰기를 포함할 경우 *''* 또는 *""* 부호로 감싸서 입력하세요. 부호로 감싼 키워드에 *\\*를 포함할 경우 *\\\\*로 입력하세요. ![검색 아이콘](/img/ic-search-on-press.svg) ***검색*** 버튼을 선택하면 필터가 적용된 데이터를 ![number-s3](/img/number-03.png) 영역에서 조회할 수 있습니다.

![필터 적용](https://img.whatap.io/media/images/livetail_003_filter.png)

#### 필터 수정

필터에 값을 입력한 뒤 입력한 값을 클릭하면 해당 값을 수정할 수 있습니다.  
![](https://img.whatap.io/media/images/livetail_005_filter_input.png)

#### 필터 수식

입력된 필터 값 아래에 있는 수식(expression)은 로그 데이터 조회 시 적용될 필터 수식 미리 보기입니다.  
![](https://img.whatap.io/media/images/livetail_006_filter_expression.png)

#### 검색 키(Search Key)

왼쪽 ![number-s](/img/number-02.png) 영역에 있는 태그는 카테고리별로 파싱(parsing) 된 검색 키입니다. 태그를 선택하여 필터를 입력할 수 있습니다. <span class='logwr'>주황색</span> 태그는 카테고리, <span class='lowfg'>파란색</span> 태그는 검색 키입니다.

예를 들어 ![number-s](/img/number-02.png) 영역의 <span class='logwr'>AppLog</span>와 <span class='logwr'>AppStdOut</span>은 카테고리, 그 아래 <span class='lowfg'>oid</span>와 같은 태그는 파싱(parsing) 된 검색 키입니다. 검색 키는 ***로그*** > ***로그 설정***의 ***로그 Parser 설정***에서 파싱 로직을 등록해 설정할 수 있습니다.

#### 필터 입력 문법

필터 입력 시 사용할 수 있는 문법 목록입니다. ***라이브 테일***과 ***로그 검색***은 동일한 문법을 사용합니다.

|문법|예시|설명|
|----|----|----|
|include(+)|+ote-gateway|ote-gateway를 포함한 것만 필터링|
|exclude(-)|-ote-gateway|ote-gateway를 제외한 것만 필터링|
|startWith(*)|ERROR*|ERROR로 시작하는 것만 필터링|
|endWith(*)|*ERROR|ERROR로 끝나는 것만 필터링|

## 테이블 설정하기

* ![number-s](/img/number-03.png) 영역 오른쪽 ***테이블 설정*** 메뉴는 ***라이브 테일***, ***로그 검색***, ***로그 트렌드***에서 사용할 수 있습니다. 
* ![아이콘](/img/ic-setting.svg) ***테이블 설정*** 버튼을 선택하면 ***컬럼 추가***와 ***테이블 컨텐츠 설정*** 옵션 메뉴가 나타납니다. 
    
    ![](https://img.whatap.io/media/images/livetail_013_table_options.png)  

    1. **컬럼 설정** 
    * ***컬럼 추가***: 태그를 선택하여 테이블에 컬럼을 추가할 수 있습니다.
    * ***컬럼 순서 설정***: 컬럼을 추가하면 컬럼 순서 설정에 해당 컬럼이 추가됩니다. 원하는 컬럼을 드래그하여 컬럼의 순서를 변경하세요.

    2. **테이블 설정**  

    ![로그 테이블 컨텐츠 설정](/img/aws-log-p/log-table-contents.png)
    * ***컨텐츠 표시 여부***
        * 체크된 항목은 테이블에 표시되지 않습니다. 기본으로 ***로그***, ***태그*** 모두 체크가 되어있으며 두 가지 항목 모두 표시합니다.
        * 다음과 같이 ***태그***를 해제할 경우 테이블에서 로그의 ***태그***는 표시되지 않습니다.
        ![](https://img.whatap.io/media/images/livetail_017_table_column_visible.png)
    * ***태그 관리***
        * 태그 관리 목록에 태그를 추가하면 추가한 순서대로 로그의 태그가 나열됩니다. 태그의 순서는 드래그하여 변경할 수 있습니다.
        * 추가한 태그를 비활성화하면 비활성화한 태그는 로그의 태그에 노출되지 않습니다.
