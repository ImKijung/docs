**피쳐 프로젝트**<ScrollToLink anchor='feature-project' />

와탭은 사용자가 원하는 소프트웨어의 모니터링 지원 여부를 쉽게 확인하고 간편하게 모니터링을 시작할 수 있도록 새롭게 **피쳐 프로젝트**(<code class='Features'>Features</code>)를 도입하였습니다. 피쳐 프로젝트는 기존 모니터링 제품을 사용자가 모니터링하고자 하는 소프트웨어에 맞게 재포장해 최적화된 모니터링 솔루션을 제공합니다. 와탭 피쳐 프로젝트를 통해 서비스의 안정성과 성능을 더욱 효과적으로 관리해 보세요. 

:::

## 주요 메뉴 알아보기 

### 맞춤형 대시보드 

대시보드는 여러 섹션으로 나뉘어 있으며, 주요 메트릭을 모니터링합니다.

1. **클러스터 상태 개요**
    - **Cluster Size**: 클러스터에 포함된 노드 수를 표시합니다.
    - **Cluster Object Counts**: 클러스터 내 객체 수를 보여줍니다.
2. **읽기/쓰기 작업**
    - **Read Ops**: 각 노드의 읽기 작업 수를 표시합니다.
    - **Write Ops**: 각 노드의 쓰기 작업 수를 표시합니다.
3. **연결 및 I/O 성능**
    - **Aerospike Cluster DB Connections**: 데이터베이스 연결 수를 모니터링합니다.
    - **IO Percent**: 각 노드의 I/O 사용량을 보여줍니다.
    - **Traffic In/Out**: 네트워크 트래픽을 모니터링합니다.
4. **리소스 사용량**
    - **CPU**: 각 노드의 CPU 사용량을 표시합니다.
    - **Memory Used Pct**: 각 노드의 메모리 사용률을 모니터링합니다.
    - **Disk Pressure**: 디스크 사용 압력을 보여줍니다.
5. **Namespace 메트릭**
    - **Namespace In-Memory Index**: 인메모리 인덱스 크기를 표시합니다.
    - **Namespace Size**: 네임스페이스 크기를 모니터링합니다.

### 주요 모니터링 메트릭

1. **CPU 사용량**: CPU 사용률을 추적하여 과부하를 방지합니다.
2. **메모리 사용량**: 메모리 사용을 모니터링하여 메모리 부족 오류를 방지합니다.
3. **스토리지 사용량**: 충분한 스토리지가 있는지 확인합니다.
4. **읽기/쓰기 작업**: 성능 병목을 나타낼 수 있는 읽기 및 쓰기 작업 수를 모니터링합니다.
5. **네트워크 트래픽**: 네트워크 사용량을 추적하여 트래픽 문제를 방지합니다.
